Prim's algorithm — full line-by-line code explanation with dry run

Core idea in one line
Start from vertex 0. Repeatedly pick the cheapest edge that connects the current tree to a new (unvisited) vertex. Input is a cost matrix — 0 means no edge (treated as ∞).
Kruskal vs Prim — quick diff
Kruskal
Works on edges. Sort all edges first. Uses Union-Find. Better for sparse graphs.
Prim
Works on vertices. Grows tree from a source. Uses cost matrix. Better for dense graphs.

Key things examiner will ask
Why two nested loops inside the outer loop? — Outer loop runs V−1 times (pick V−1 edges). Inner double loop scans ALL visited→unvisited pairs to find the minimum cost edge each time.
Why initialize cost[i][j] = INF when input is 0? — 0 means no direct edge. If we keep it 0, it would always be picked as minimum, which is wrong.
Time complexity? — Three nested loops → O(V²). This is why Prim suits dense graphs (E ≈ V²).
What's visited[] for? — Tracks which vertices are already in the MST. We only look at edges from visited→unvisited to avoid re-adding vertices.
Does Prim guarantee MST? — Yes. Greedy choice property holds — at each step the locally optimal edge is also globally optimal for spanning trees.
