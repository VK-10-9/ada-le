Kruskal's algorithm — full line-by-line code explanation with dry run

Core idea in one line
Sort all edges by weight → greedily pick cheapest → skip if it creates a cycle → stop at V−1 edges. Use Union-Find to detect cycles in O(1).

Union-Find — how cycle detection works
Each node starts as its own parent. find(x) climbs to the root. unite(x,y) merges two trees. If find(u) == find(v) → same set → cycle!
Initial parent[] (each node = own parent):

parent[0]=0

Key things examiner will ask
Why qsort? — We need edges sorted by weight. qsort uses comparator function that returns difference of weights.
Why path compression in find()? — Without it, find() is O(n). With it, nearly O(1). Makes Union-Find efficient.
Time complexity? — O(E log E) for sorting. Union-Find is O(α(V)) ≈ O(1). Total: O(E log E).
When to stop? — When V−1 edges are picked. The loop runs over all E edges but exits implicitly after enough picks.
Kruskal vs Prim? — Kruskal works on edges (good for sparse graphs). Prim works on vertices via adjacency matrix (good for dense graphs).
