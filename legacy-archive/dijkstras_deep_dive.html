Dijkstra's algorithm — full line-by-line explanation with dry run

Core idea in one line
From a source, greedily pick the closest unvisited vertex, mark it done, then relax (update) distances to all its unvisited neighbors. Repeat V−1 times.
Floyd vs Dijkstra — key diff
Floyd's (Lab 3a)
All-pairs shortest path. Every vertex to every other vertex. O(n³). Works with negative weights (no negative cycles).
Dijkstra's (Lab 4)
Single-source shortest path. One source → all others. O(V²). Does NOT work with negative weights.

Viva questions
Why doesn't Dijkstra work with negative weights? — Once a vertex is marked visited, its distance is locked. A negative edge discovered later could give a shorter path, but we've already finalized it. Use Bellman-Ford for negative weights.
Why dist[src]=0 separately after init? — We copy the source row into dist[], which sets dist[src][src] = 0 (or INF after conversion). Setting dist[src]=0 explicitly ensures source distance is always 0.
Why loop runs n−2 times (c < n−1)? — Source is already visited. We need to finalize n−1 more vertices. But inner loop handles last vertex implicitly, so n−2 iterations suffice.
Time complexity? — Two nested loops (pick min + relax neighbors) → O(V²). Same as Prim's — both use adjacency matrix + greedy vertex picking.
Dijkstra vs Prim — they look similar? — Yes, same structure. Prim picks minimum edge weight to grow MST. Dijkstra picks minimum cumulative distance from source. Different goal, same skeleton.
