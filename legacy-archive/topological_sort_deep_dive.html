Topological Sort using Kahn's algorithm — full line-by-line explanation with dry run

Core idea in one line
Find all vertices with no incoming edges (indegree=0), process them first, reduce neighbors' indegree. Repeat. If all vertices aren't processed → cycle exists → topological sort impossible.
What is indegree?
Indegree of a vertex = number of edges coming INTO it.
Vertex with indegree 0 = no prerequisites = can be scheduled first.
Example: In a course prereq graph, "Maths" (no prereqs) has indegree 0. "ADA" (needs Maths) has indegree ≥ 1.

Cycle detection — how it works
If a cycle exists (e.g. 1→2→3→1), those vertices never reach indegree 0 — they never enter the queue — they never get counted.
After the loop: if (count != n) → some vertices were never processed → cycle detected → print error.
Viva questions
What is topological ordering? — A linear ordering of vertices such that for every directed edge u→v, u appears before v. Only possible on a DAG (Directed Acyclic Graph).
Why does Kahn's use a queue (BFS)? — All indegree-0 vertices are equally "ready" to be processed. Queue gives FIFO order — process them level by level, like BFS traversal.
Is topological order unique? — No. Multiple valid orderings can exist if multiple vertices have indegree 0 at the same time. E.g. both 0→1→3 and 0→2→3 are valid.
Time complexity? — O(V+E). Every vertex is enqueued/dequeued once (O(V)), every edge is processed once when reducing indegree (O(E)).
Why not possible on undirected graphs? — Topological sort requires direction — it defines "what comes before what." Undirected edges have no such ordering.
Real-world use? — Build systems (compile order), course prerequisites, task scheduling, package dependency resolution.
