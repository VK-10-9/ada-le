Floyd's and Warshall's algorithms — full line-by-line explanation with dry runs

The key insight — same skeleton, different operation
Floyd's — Lab 3a
Finds shortest distances between all pairs. Uses min(dist[i][j], dist[i][k]+dist[k][j]). Output: distance matrix.
Warshall's — Lab 3b
Finds reachability between all pairs. Uses g[i][j] || (g[i][k] && g[k][j]). Output: 0/1 matrix.
Both use the same triple nested loop: for k → for i → for j. Time: O(n³). Space: O(n²). The only difference is the update formula inside.

How k=0 works — example trace
k=0 means: "Can we go through vertex 0 to get a shorter path?"
Check dist[1][3]: currently ∞. Via 0: dist[1][0] + dist[0][3] = ∞ → no update.
Check dist[3][1]: currently ∞. Via 0: dist[3][0] + dist[0][1] = 2 + 3 = 5 → update!
This continues for all k from 0 to n−1.
Viva questions
Why k is the outermost loop? — When considering k as intermediate, dist[i][k] and dist[k][j] must already be finalized. Outer k ensures all shorter paths via 0..k−1 are already computed before trying k.
Does it work with negative weights? — Yes, but NOT with negative cycles. If dist[i][i] becomes negative after running, there's a negative cycle.
Time/Space complexity? — Time: O(n³). Space: O(n²) for the matrix.
What's in dist[i][i] at start? — 0. Distance from a vertex to itself is always 0.

How the OR/AND formula works
g[i][j] = g[i][j] || (g[i][k] && g[k][j])

In English: "Can i reach j directly, OR can i reach k AND k reach j?"
If either is true → set g[i][j] = 1 (reachable).
Example (k=1): g[0][2] = g[0][2] || (g[0][1] && g[1][2]) = 0 || (1 && 1) = 1 ✓
Viva questions
Floyd vs Warshall — exact difference? — Floyd finds shortest DISTANCES (actual weights). Warshall finds REACHABILITY (yes/no). Floyd uses min/+, Warshall uses OR/AND.
What does g[i][j]=1 mean? — There exists at least one path from vertex i to vertex j (of any length). It does NOT give you the path or its cost.
Input for Warshall? — Simple adjacency matrix (0/1). No weights needed. No INF handling needed.
Time complexity? — O(n³) same as Floyd. Three nested loops.
